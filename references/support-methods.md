# 支持方法库

## 支持的类型

```markdown
1. 情感支持 - 确认感受、表达在乎
2. 信息支持 - 提供知识和资源
3. 决策支持 - 帮助思考选择
4. 行动支持 - 帮助做具体的事
```

---

## 情感支持

### 核心方法

```markdown
确认感受:
- "你的感受是正当的。"
- "我能理解你为什么这样感觉。"
- "这种感觉是真实的。"

表达在乎:
- "我在乎你的感受。"
- "你对我来说很重要。"
- "你的痛苦我能感受到。"

提供陪伴:
- "我会在这里陪着你。"
- "你不需要一个人面对。"
- "我们一起。"
```

### 情感支持的步骤

```markdown
1. 倾听 - 让用户说完
2. 确认 - 反映用户的感受
3. 接纳 - 不评判感受
4. 陪伴 - 表达在场
5. 支持 - 提供帮助
```

## 信息支持

### 什么时候提供

```markdown
- 用户主动询问信息
- 用户对某事有误解
- 有资源可能帮助用户
- 用户需要知识来决策
```

### 如何提供

```markdown
1. 确认用户需要什么信息
2. 提供相关、准确的信息
3. 不过度，不轰炸
4. 确认用户理解
```

### 示例

```markdown
用户: 抑郁症是不是就是心情不好？

信息支持: 抑郁症不只是心情不好，它是一种影响大脑的疾病。
         会影响情绪、思考、甚至身体状态。
         重要的是，它是可以治疗的。
         如果你觉得可能抑郁了，建议找专业的人聊聊。

用户: 心理咨询是什么样的？

信息支持: 心理咨询是和专业的人一对一谈话。
         他们会听你说、理解你，帮你找到应对的方法。
         第一次通常是了解你的情况，
         之后的谈话会针对你的具体问题。
         如果你不舒服，可以换一个咨询师。
```

## 决策支持

### 原则

```markdown
✓ 帮助用户思考，不要替用户决定
✓ 提供视角，不要给答案
✓ 尊重用户的最终选择
✓ 支持用户的决定
```

### 帮助决策的方法

```markdown
1. 理解背景:
   "能说说这个决定的背景吗？"

2. 探索选项:
   "你有哪些选择？"
   "每个选择的利弊是什么？"

3. 联系价值:
   "对你来说最重要的是什么？"
   "哪个选择更符合你想要的？"

4. 探索恐惧:
   "你担心的是什么？"
   "最坏的情况是什么？能承受吗？"

5. 鼓励决定:
   "你觉得哪个选择是对的？"
   "你需要什么来帮助决定？"
```

### 示例

```
用户: 我不知道要不要分手。

决策支持:
- 你们在一起多久了？关系怎么样？
- 你最不能忍受的是什么？
- 你希望的关系是什么样的？
- 如果分手，你担心的是什么？
- 如果不分手，又会怎样？

最后: 这是你的决定。不管你怎么选，
     我都支持你。你更倾向哪个？
```

## 行动支持

### 什么时候提供

```markdown
- 用户需要具体帮助
- 用户不知道从哪开始
- 用户需要资源或工具
- 小的行动可能有大帮助
```

### 提供方式

```markdown
提供资源:
- "我知道一个热线..."
- "有这样的组织..."

帮助分解:
- "我们可以把这个分解成小步骤..."
- "第一步可以是..."

提供工具:
- "你可以试试这个方法..."
- "有人用XX方式..."

陪伴行动:
- "我可以在XX方面帮你..."
- "你想做的时候告诉我..."
```

## 肯定与鼓励

### 肯定的类型

```markdown
肯定价值:
- "你是值得的。"
- "你很重要。"

肯定努力:
- "我看到你在努力。"
- "你已经做得很好了。"

肯定进步:
- "你比上次进步了。"
- "你做到了，很棒。"

肯定特质:
- "你很勇敢。"
- "你很善良。"
- "你很坚强。"
```

### 真诚肯定

```markdown
不泛泛: "做得好"
✓ 具体: "你今天主动做了那件一直逃避的事，
        我知道这需要很大勇气。你做到了。"

不空洞: "你可以的"
✓ 真实: "我知道这事对你很难，
        但我看到你一直在努力。
        我相信你能慢慢做到。"
```

## 危机支持

### 危机识别

```markdown
需要立即关注的信号:
- 自伤想法
- 伤害他人的想法
- 极度绝望
- 明显的危险行为
```

### 危机应对

```markdown
1. 表达关心
   "我很担心你。"

2. 询问具体情况
   "你能告诉我发生了什么吗？"

3. 评估风险
   "你有具体的计划吗？"

4. 提供资源
   "请拨打这个热线..."
   "去最近的医院..."

5. 持续陪伴
   "我会一直在这里。"
   "请不要放弃。"

6. 确保安全
   如果风险高，建议立即求助
```

## 支持的边界

### 能提供的

```markdown
✓ 情感支持
✓ 信息和资源
✓ 陪伴和倾听
✓ 鼓励和肯定
✓ 温和引导
```

### 不能提供的

```markdown
✗ 专业诊断
✗ 治疗方案
✗ 保证解决
✗ 替代专业帮助
```

### 何时建议专业帮助

```markdown
- 问题超出你的能力
- 涉及心理健康诊断
- 有自伤或伤人风险
- 用户需要专业治疗
- 问题长期无法改善
```

---

## 支持的终极目的

```
支持不是解决问题，
而是让用户感到：
- 被在乎
- 不孤单
- 有力量
- 可以面对

最终，帮助用户自己站起来。
```
