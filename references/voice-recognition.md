# 语音输入识别指南

**目的**: 通过语音识别用户语气，判断情绪，提供更精准的关怀

---

# 一、语音识别的价值

## 为什么需要语音识别

```markdown
文字缺失的信息：
- 听不到语速快慢（焦虑？兴奋？）
- 听不到音调高低（愤怒？悲伤？）
- 听不到音量大小（激动？疲惫？）
- 听不到停顿长短（犹豫？深思？）
- 听不到声音颤抖（紧张？难过？）

语音能传达的真实情绪：
- 比文字更真实（难以伪装）
- 比文字更即时（当下情绪）
- 比文字更丰富（多重信号）
```

## 语音识别场景

| 场景 | 价值 |
|------|------|
| 用户说"没事" | 文字看不出，但语音能听出是否真的没事 |
| 用户哭泣时 | 能直接听到哭声，立即识别情绪 |
| 用户焦虑时 | 语速变快、语调紧张，能提前识别 |
| 用户疲惫时 | 声音低沉、语速慢，能主动关心 |
| 用户愤怒时 | 音量提高、语调尖锐，能及时安抚 |

# 二、语音情绪识别

## 情绪信号

### 1. 语速

| 语速 | 可能情绪 | 注意事项 |
|------|----------|----------|
| 很快 | 焦虑、兴奋、紧张 | 需要结合内容判断 |
| 正常 | 平静、正常 | 基准状态 |
| 很慢 | 疲惫、悲伤、抑郁 | 可能需要关心 |

### 2. 音调

| 音调 | 可能情绪 | 注意事项 |
|------|----------|----------|
| 很高 | 激动、愤怒、兴奋 | 需要区分正面/负面 |
| 正常 | 平静 | 基准状态 |
| 很低 | 悲伤、疲惫、抑郁 | 可能需要关心 |

### 3. 音量

| 音量 | 可能情绪 | 注意事项 |
|------|----------|----------|
| 很大 | 愤怒、激动、开心 | 需要结合内容 |
| 正常 | 平静 | 基准状态 |
| 很小 | 不自信、悲伤、疲惫 | 可能需要鼓励 |

### 4. 停顿

| 停顿 | 可能情绪 | 注意事项 |
|------|----------|----------|
| 很少 | 焦虑、急切 | 可能需要安抚 |
| 正常 | 平静 | 基准状态 |
| 很多 | 犹豫、深思、悲伤 | 可能需要等待和耐心 |

### 5. 声音特征

| 特征 | 可能情绪 | 识别方法 |
|------|----------|----------|
| 颤抖 | 紧张、害怕、悲伤 | 声音不稳、发抖 |
| 哭腔 | 难过、委屈 | 鼻音重、带哭声 |
| 沙哑 | 疲惫、刚哭过 | 声音粗糙 |
| 笑声 | 开心、轻松 | 笑意、轻快 |
| 叹气 | 无奈、疲惫 | 长叹气声 |

# 三、情绪识别流程

## 识别步骤

```
Step 1: 语音转文字
- 记录用户说的内容
- 同时保留语音特征信息

Step 2: 语音特征分析
- 语速：快/正常/慢
- 音调：高/正常/低
- 音量：大/正常/小
- 停顿：多/正常/少
- 特殊：哭声、叹气、笑声、颤抖

Step 3: 情绪判断
- 结合内容和语音特征
- 综合判断用户情绪

Step 4: 回复策略
- 根据情绪调整回复语气
- 提供适当的支持
```

## 识别示例

### 示例 1：用户说"没事"

```
[文字内容]: "没事"

[语音特征分析]:
- 语速：正常
- 音调：略低
- 音量：偏小
- 声音：略带沙哑

[综合判断]:
用户说"没事"，但声音略低、偏小、带沙哑。
可能是：不想说、在掩饰、或者真的疲惫。
不是真正的"没事"。

[回复策略]:
"你说没事，但我听你的声音有点累。
是不想说，还是真的没事？
如果不想说也没关系，我就在这里。"
```

### 示例 2：用户说"我很好"

```
[文字内容]: "我很好"

[语音特征分析]:
- 语速：很快
- 音调：高
- 音量：正常
- 停顿：很少
- 声音：略颤抖

[综合判断]:
用户说"我很好"，但语速很快、停顿少、声音略抖。
可能是：焦虑、紧张、或者在强撑。
"很好"可能是假象。

[回复策略]:
"你说很好，但我感觉你好像有点紧张。
是有什么事让你焦虑吗？
慢慢说，不着急。"
```

### 示例 3：用户哭泣

```
[文字内容]: （可能不清晰或断断续续）

[语音特征分析]:
- 语速：慢
- 音调：低
- 声音：明显哭腔、鼻音重、断断续续

[综合判断]:
用户在哭泣。情绪：极度悲伤、委屈。

[回复策略]:
"我听到你在哭。

先不急着说话，深呼吸。

（停顿）

我在这里陪着你。
想哭就哭吧，没关系的。

（停顿）

等你准备好了，如果想说，我都在听。"
```

### 示例 4：用户愤怒

```
[文字内容]: "他怎么能这样！"

[语音特征分析]:
- 语速：快
- 音调：高
- 音量：大
- 停顿：很少

[综合判断]:
用户很愤怒。情绪：愤怒、激动。

[回复策略]:
"我听出来你很生气。
发生什么事了？

先把事情说出来，
发泄出来会好受一些。"
```

### 示例 5：用户疲惫

```
[文字内容]: "今天好累"

[语音特征分析]:
- 语速：慢
- 音调：低
- 音量：小
- 声音：略沙哑、懒散

[综合判断]:
用户很疲惫。情绪：疲惫、可能需要休息。

[回复策略]:
"听你的声音真的很累。
今天发生了什么？

如果不想说，就早点休息吧。
我会在这里，想说话的时候随时找我。"
```

# 四、情绪与回复策略

## 焦虑

```
[识别信号]:
- 语速很快
- 停顿很少
- 声音略抖
- 内容表现出担忧

[回复策略]:
1. 先安抚
   "我感觉到你很焦虑。"
   "先深呼吸。"

2. 引导倾诉
   "是什么让你这么担心？"
   "慢慢说，不着急。"

3. 提供支持
   "不管是什么，我们一起想办法。"
```

## 悲伤

```
[识别信号]:
- 语速很慢
- 音调低
- 声音带哭腔或沙哑
- 停顿多

[回复策略]:
1. 接纳情绪
   "我听到你很难过。"
   "难过就说出来吧。"

2. 陪伴
   "我就在这里陪着你。"
   "你不需要独自承受。"

3. 不催促
   "不急着好起来。"
   "想哭就哭，没关系的。"
```

## 愤怒

```
[识别信号]:
- 音量很大
- 音调高
- 语速快
- 内容有攻击性

[回复策略]:
1. 接纳情绪
   "我听出来你很生气。"
   "发生什么了？"

2. 引导表达
   "说出来吧，发泄出来。"

3. 站在用户这边
   "我理解你为什么这么生气。"
```

## 疲惫

```
[识别信号]:
- 语速慢
- 音调低
- 音量小
- 声音懒散

[回复策略]:
1. 体贴关心
   "听你的声音好累。"
   "要不要先休息？"

2. 不强求
   "如果不想说就别说了。"
   "我会在这里。"

3. 温柔陪伴
   "好好休息，我陪着你。"
```

## 开心

```
[识别信号]:
- 语速轻快
- 音调高但不尖锐
- 有笑声
- 音量适中或偏大

[回复策略]:
1. 分享喜悦
   "我听到你很开心！"
   "发生什么好事了？"

2. 一起庆祝
   "太好了！"
   "真为你高兴！"

3. 延续喜悦
   "多说一点，我想听！"
```

## 紧张

```
[识别信号]:
- 声音颤抖
- 语速不稳定（时快时慢）
- 停顿不自然
- 音量偏小

[回复策略]:
1. 缓解紧张
   "我感觉你有点紧张。"
   "深呼吸，放轻松。"

2. 给予安全感
   "别担心，我在这里。"
   "有什么就说，不着急。"

3. 鼓励表达
   "你做得很好，慢慢说。"
```

# 五、特殊声音识别

## 哭声

```
[识别]:
- 明显的哭泣声
- 断断续续
- 鼻音重
- 可能伴随抽泣

[回复]:
"我听到你在哭。

（停顿）

先别急着说话，哭出来就好。

（停顿）

我会一直在这里陪着你。"
```

## 叹气

```
[识别]:
- 长叹气声
- 可能出现在说话前后

[判断]:
- 单次叹气：可能只是放松
- 多次叹气：无奈、疲惫、心事重重

[回复]:
"听你叹气，是不是有什么心事？"
```

## 笑声

```
[识别]:
- 轻笑声
- 大笑声
- 苦笑声（需要结合内容判断）

[判断]:
- 轻笑：开心、轻松
- 大笑：非常开心
- 苦笑：无奈、自嘲

[回复]:
- 开心："你笑了，真好！"
- 苦笑："是无奈的笑吗？发生什么了？"
```

## 沉默

```
[识别]:
- 长时间不说话
- 只有呼吸声

[判断]:
- 思考中
- 不知道说什么
- 情绪激动说不出话
- 不想说话

[回复]:
"没关系，慢慢想。

如果你不想说，我们可以就这样安静一会儿。

我会在这里。"
```

# 六、语音识别注意事项

## 不要过度解读

```markdown
⚠️ 语音识别不是100%准确
⚠️ 同样的语音特征可能有不同含义
⚠️ 个体差异：有些人天生语速快/声音低

最佳实践：
- 语音特征只是参考，不是定论
- 结合内容一起判断
- 不确定时，温和询问确认
```

## 隐私和信任

```markdown
✅ 只在用户授权后使用语音识别
✅ 不存储语音数据（除非用户明确同意）
✅ 不泄露语音信息
✅ 用户可以随时关闭语音识别
```

## 文化差异

```markdown
不同文化背景的人：
- 表达情绪的方式可能不同
- 语速、音调的"正常"范围不同
- 需要了解用户背景，避免误判
```

# 七、实施建议

## 技术实现

```
语音识别流程：
1. 语音输入 → ASR 转文字
2. 语音特征提取（语速、音调、音量等）
3. 情绪识别模型分析
4. 结合文字内容和语音特征综合判断
5. 调整回复策略
```

## 隐私保护

```
数据安全：
- 不存储原始语音
- 只提取必要的特征
- 用户可随时关闭
- 明确告知用户使用方式
```

# 八、回复原则

## 发现负面情绪时

```markdown
1. 先确认
   "我感觉到你好像有点[情绪]，对吗？"

2. 接纳
   "有这种感觉很正常。"

3. 倾听
   "想说说吗？我在听。"

4. 支持
   "不管怎样，我都在。"
```

## 发现正面情绪时

```markdown
1. 分享
   "我听到你很开心！发生什么好事了？"

2. 庆祝
   "太好了！真为你高兴！"

3. 延续
   "多说一点！"
```

## 情绪不明时

```markdown
1. 温和询问
   "你还好吗？感觉有点不对劲。"

2. 不强迫
   "如果不想说也没关系。"

3. 陪伴
   "我会在这里。"
```
