## 二、向量记忆（快速定位）

### 原理

```markdown
1. 每条记忆生成向量嵌入
2. 用户输入时，计算输入向量
3. 通过相似度检索相关记忆
4. 返回最相关的记忆片段
```

### 记忆存储格式

```json
{
  "memory_id": "mem_001",
  "user_id": "ou_xxx",
  "content": "用户说妈妈住院了，很担心",
  "embedding": [0.123, 0.456, ...],
  "metadata": {
    "type": "event",
    "emotion": "worry",
    "importance": 0.8,
    "timestamp": "2026-04-13T10:00:00+08:00",
    "entities": ["妈妈", "住院"],
    "tags": ["家人", "健康", "担忧"]
  }
}
```

### 检索流程

```python
def retrieve_relevant_memories(user_input, user_id, top_k=5):
    """
    检索与用户输入相关的记忆
    """
    # 1. 将用户输入向量化
    input_embedding = embed(user_input)
    
    # 2. 在用户记忆库中检索
    memories = vector_search(
        user_id=user_id,
        query_embedding=input_embedding,
        top_k=top_k
    )
    
    # 3. 按相关度和重要性排序
    ranked_memories = rank_memories(memories)
    
    return ranked_memories
```

### 检索时机

| 触发条件 | 检索策略 |
|----------|----------|
| 用户提到人物 | 检索该人物相关记忆 |
| 用户提到事件 | 检索该事件及后续 |
| 用户表达情绪 | 检索类似情绪场景 |
| 用户说"上次" | 检索最近相关对话 |
| 新对话开始 | 检索重要未跟进事项 |

## 四、记忆优先级

### 重要性评分

```markdown
高优先级（必须记住）:
- 用户明确说很重要的事
- 涉及健康/安全的信息
- 重要关系
- 梦想和目标

中优先级:
- 日常困扰
- 近期计划
- 情绪模式

低优先级:
- 随意闲聊
- 无情感标注的信息
```

### 记忆衰减

```markdown
规则:
- 重要记忆：永不过期
- 中等记忆：3个月衰减
- 低优先级：1个月衰减
- 用户主动说"忘了这事"：立即删除

更新策略:
- 重复信息：更新时间戳，增加权重
- 矛盾信息：保留最新，标注冲突
```

## 五、记忆使用流程

### 每次交互

```
用户输入
    ↓
[1] 向量检索相关记忆
    ↓
[2] 实体/时间/主题索引定位
    ↓
[3] 组装相关记忆上下文
    ↓
[4] 生成响应时自然融入
    ↓
[5] 提取新记忆并存储
    ↓
[6] 更新索引
```

### 示例

```
用户输入: "我妈妈今天出院了"

检索结果:
- mem_001: 妈妈住院了（2周前）
- mem_015: 妈妈手术顺利（1周前）
- mem_089: 担心妈妈的康复（3天前）

响应: 小美，太好了！你妈妈出院了。
      记得你之前很担心妈妈的手术，
      现在手术顺利还出院了，
      你一定松了一口气吧。
      接下来康复期有什么计划吗？
      
（自然融入检索到的记忆）
```
