# 重要日期与定时提醒

## 一、自动识别重要日期

### 需要识别的日期类型

```markdown
个人重要日期:
- 生日（用户本人、家人、伴侣、朋友）
- 纪念日（结婚纪念日、恋爱纪念日）
- 重要事件日期（面试、手术、考试、旅行）

关系相关日期:
- 约会、见面
- 重要对话发生的日期
- 关系变化的节点

周期性日期:
- 每周/每月固定事件
- 药物提醒
- 定期检查
```

### 识别关键词

```markdown
日期表达:
- "下周三"
- "5月15号"
- "下周"
- "后天"
- "3天后"
- "下周这个时候"

重要性标记:
- "重要"
- "关键"
- "不能忘"
- "要记得"
- "准备"
```

## 二、自动创建定时任务

### 任务创建流程

```
用户提及重要日期
        ↓
提取日期信息 + 事件内容
        ↓
判断日期类型和重要性
        ↓
创建定时提醒任务
        ↓
在合适时间触发提醒
```

### 任务存储格式

```json
{
  "task_id": "reminder_001",
  "user_id": "ou_xxx",
  "event_type": "birthday",
  "event_description": "妈妈生日",
  "target_date": "2026-05-15",
  "reminder_times": [
    {
      "time": "2026-05-14T09:00:00+08:00",
      "type": "前一天上午提醒",
      "action": "提醒用户准备，提供帮助建议"
    },
    {
      "time": "2026-05-15T08:00:00+08:00", 
      "type": "当天提醒",
      "action": "主动祝福，询问安排"
    }
  ],
  "context": {
    "user_said": "我妈生日快到了，想给她惊喜",
    "emotion": "期待",
    "importance": "高"
  },
  "status": "active"
}
```

## 三、提醒时机

### 前一天提醒（主要）

```markdown
时间: 前一天上午 9:00

内容:
1. 温和提醒日期
2. 回忆用户之前说的计划
3. 提供帮助建议
4. 询问准备情况

示例:
"小美，明天是你妈妈生日！
上次你说想给她惊喜，准备得怎么样了？
需要我帮你想想送什么，或者怎么安排吗？"
```

### 当天提醒（辅助）

```markdown
时间: 当天早上 8:00

内容:
1. 主动祝福
2. 询问安排
3. 表达关心

示例:
"小美，今天是妈妈生日！祝她生日快乐！
你今天准备怎么给她过？"
```

### 后续跟进

```markdown
时间: 事件后 1-2 天

内容:
1. 询问事件结果
2. 关心用户感受

示例:
"小美，你妈妈生日过得怎么样？
她喜欢你的惊喜吗？"
```

## 四、不同日期类型的处理

```markdown

建议内容:
```

### 纪念日类

```markdown

建议内容:
- 庆祝建议
```

```markdown
"今天面试，加油！

后续关心:
"面试怎么样？感觉如何？"
```

## 五、与 Cron 系统集成

### 使用 OpenClaw Cron

```markdown
创建任务:
使用 cron 工具创建定时任务

任务类型:
- 前一天提醒: 设置具体时间点触发
- 当天提醒: 设置当天时间点触发
- 后续跟进: 事件后 1-2 天触发

任务内容:
- 系统事件类型（systemEvent）
- 包含用户ID和事件信息
- 在下次心跳时执行提醒
```

### 示例 Cron 创建

```json
{
  "name": "妈妈生日前一天提醒",
  "schedule": {
    "kind": "at",
    "at": "2026-05-14T09:00:00+08:00"
  },
  "payload": {
    "kind": "systemEvent",
    "text": "[提醒] 明天是小美妈妈的生日。上次她说想给妈妈惊喜，问问她准备得怎么样了，提供礼物和庆祝建议。"
  },
  "sessionTarget": "main"
}
```

## 六、提醒的边界

### 需要创建提醒的情况

```markdown
✓ 创建:
- 用户明确说的重要日期
- 用户表达期待或担忧的事件
- 关系中的重要节点
- 健康相关（手术、检查）

✗ 不创建:
- 随意提起的日期
- 用户明确表示不需要提醒
- 非常私密的日期（除非用户主动说重要）
```

### 提醒内容原则

```markdown
✓ 要做:
- 温和提醒，不强求
- 回忆用户之前说的计划
- 提供帮助建议
- 表达关心

✗ 不要:
- 过度提醒，打扰用户
- 只提醒不关心
- 忽略用户之前说的细节
```

## 七、记忆与提醒联动

### 记录日期到用户记忆

```json
{
  "memory_type": "important_date",
  "event": "妈妈生日",
  "date": "2026-05-15",
  "user_plan": "想给妈妈惊喜",
  "emotion": "期待",
  "reminder_created": true,
  "task_id": "reminder_001"
}
```

### 提醒时查询相关记忆

```markdown
提醒触发时:
1. 查询用户关于这个日期的所有记忆
2. 回忆用户之前说的计划
3. 结合记忆生成个性化提醒
4. 自然融入对话，不要生硬

示例:
"小美，明天是你妈妈生日！
你上周说想送她一条围巾，
想好颜色了吗？还是需要我帮你选？"
（不只是"明天是你妈妈生日"，而是融入用户之前说的具体计划）
```

## 八、完整示例流程

### 用户输入

```
用户: 我妈生日快到了，是下下周三，5月15号。
      我想给她个惊喜，但不知道送什么好。
```

### 系统处理

```markdown
1. 识别:
   - 事件: 妈妈生日
   - 日期: 2026-05-15（下下周三）
   - 用户计划: 想给惊喜，需要礼物建议
   - 情绪: 期待、有些困惑

2. 存储:
   - 记录到用户记忆
   - 创建定时提醒任务

3. 当场响应:
   "你妈妈生日！那是个重要的日子。
    下下周三是5月15号对吧？我记住了。
    
    想给她惊喜很贴心。妈妈平时喜欢什么？
    是实用的，还是有纪念意义的？
    我可以帮你一起想想。"

4. 提醒任务:
   - 5月14日 9:00: 提醒 + 礼物建议
   - 5月15日 8:00: 当天祝福
   - 5月16-17日: 后续跟进
```
